# Описание микросервисов

## 1. telegram-bot

**Назначение**  
Единственная точка взаимодействия с пользователем в Telegram.

**Технологии**  
- Python 
- aiogram   
- asyncio

**Основные функции**  
- Обработка команд: `/start`, `/profile`, `/settings`, `/help`  
- Машина состояний (FSM) для:  
  - регистрации новой анкеты  
  - редактирования анкеты (возраст, пол, город, био, интересы)  
  - пошаговой загрузки фотографий  
  - просмотра анкет других пользователей   
- Inline- и reply-клавиатуры:  
  - «Лайк» / «Пропустить»  
  - «Редактировать профиль»  
  - «Следующая»  
  - меню после мэтча  
- Отправка уведомлений:  
  - «У вас новый матч!»  
  - «Вам написали сообщение»  
  - «Ваша анкета просмотрена N раз» 

**Взаимодействие с другими сервисами**  
- HTTP-запросы к `backend` rest 
- Примеры:  
  - POST `/users/register-or-update`  
  - GET `/profiles/next`  
  - POST `/interactions/like`  
  - POST `/interactions/pass`  
  - PATCH `/profiles/my`  
  - POST `/photos/upload` (multipart)

**Особенности**  
- Обязательная обработка ошибок и fallback-сообщений  
- Middleware: logging, rate-limit, антифлуд  
- Хранение временного состояния в FSMContext  
- Скачивание и отправка файлов Telegram → backend

## 2. backend (основной API)

**Назначение**  
Центральный сервис бизнес-логики, профилей и большинства операций.

**Технологии**  
- FastAPI  
- SQLAlchemy asyncpg  
- Pydantic   
- aiohttp / httpx вызовы
- boto3 / minio-py 

**Основные роуты / модули**

- **/users/**  
  - POST `/register-or-update` — регистрация/обновление по telegram_id  
  - GET `/me` — информация о текущем пользователе  

- **/profiles/**  
  - GET `/my` — моя анкета  
  - PATCH `/my` — редактирование анкеты  
  - POST `/my/photos` — загрузка фотографии (minio)  
  - DELETE `/my/photos/{photo_index}`  
  - GET `/next` — получить следующую анкету для показа  

- **/interactions/**  
  - POST `/like` — лайк анкеты  
  - POST `/pass` — пропуск  
  - POST `/report` — жалоба 

- **/matches/**  
  - GET `/my` — список текущих мэтчей  

**Ключевые внутренние сервисы**  
- `ProfileService` — CRUD, валидация, расчёт completeness_score  
- `MatchingService` — выбор кандидатов (фильтры + рейтинг + исключение уже просмотренных)  
- `PhotoService` — загрузка, получение URL, удаление  
- `InteractionService` — запись лайков, проверка на матч, публикация события в rabbit

**Особенности**  
- Аутентификация: заголовок `Telegram-ID` или JWT  
- Кэширование предзагруженных анкет в Redis   
- Background tasks для некритичных операций  
- Логирование + метрики (Prometheus)

## 3. rating-engine

**Назначение**  
Расчёт и обновление всех уровней рейтинга анкет.

**Технологии**  
- FastAPI
- Celery   
- Redis 
- RabbitMQ



**Основные задачи Celery**

- Создание\ обновление анкеты 
- Лайк\ пропуск   
- Переодический пересчет рейтингов 
- Выборка n-анкет для redis 

**Особенности**  
- Отдельная таблица `user_ratings`  
- Оптимистическая блокировка (version column)  
- Кэширование промежуточных результатов в Redis

## 4. storage (MinIO)

**Назначение**  
Объектное хранилище для фотографий пользователей.

**Технологии**  
- MinIO (S3-совместимый сервер)  
- boto3 / minio python client (с backendа)

**Основные операции**  
- PUT — загрузка фото (от backend)  
- GET — presigned URL для временного доступа (кол-во дней)  
- DELETE — удаление старых/ненужных фото  
- Бакет: `photos` или `avatars`

**Структура ключей** 
- users/{telegram_id}/photo_1.jpg
